# TDD Implementation Skill

## 목적
기대 동작을 먼저 고정하고 변경 안정성을 높인다.

## 원칙
- 가능하면 테스트를 먼저 작성한다.
- 실패를 확인한다.
- 최소 구현으로 통과시킨다.
- 통과 후 리팩터링한다.
- 완료 시 실행한 검증을 요약한다.

## 절차
1. 원하는 동작을 한 문장으로 정의한다.
2. 기존 테스트가 있으면 보강한다.
3. 가능하면 테스트를 먼저 쓴다.
4. 실패하는지 확인한다.
5. 최소 구현을 한다.
6. 테스트를 재실행해 통과를 확인한다.
7. 관련 린트/빌드/통합 검증을 수행한다.

## 예외
아래는 테스트 선작성 없이 진행 가능하다.
- 단순 문구 변경
- 매우 작은 UI 수정
- 설정 파일 변경
- 레거시 구조상 테스트 비용이 과도한 경우

단, 이 경우에도 검증 방법과 이유는 남긴다.
