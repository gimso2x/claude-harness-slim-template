# Next.js Guidelines

## 목적
Next.js 프론트엔드 변경을 단순하고 예측 가능하게 유지한다.

## 기본 원칙
- 가능하면 서버 컴포넌트를 우선한다.
- 클라이언트 컴포넌트는 꼭 필요한 곳만 사용한다.
- 데이터 가져오기는 서버 쪽에 최대한 가깝게 둔다.
- 페이지 파일은 얇게, 복잡한 로직은 별도 함수/컴포넌트로 분리한다.

## 규칙
- "use client"는 최소화한다.
- 서버/클라이언트 경계를 명확히 한다.
- 에러/로딩 상태를 고려한다.
- 접근성 속성, 버튼 라벨, 폼 상태를 확인한다.
- UI 수정 시 작은 시각 변경이라도 영향 범위를 점검한다.

## 변경 체크리스트
- [ ] 서버 컴포넌트로 충분한가?
- [ ] 불필요한 상태/이펙트가 없는가?
- [ ] 로딩/에러 처리 고려했는가?
- [ ] 접근성 문제가 없는가?
- [ ] 필요한 경우 화면 검증 또는 스냅샷을 했는가?
